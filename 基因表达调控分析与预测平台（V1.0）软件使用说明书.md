# 基因表达调控分析与预测平台（V1.0）软件使用说明书

---

## 第一章 前言

### 1.1 编写目的

本使用说明书旨在为用户提供基因表达调控分析与预测平台的详细操作指南，帮助用户快速掌握平台各项功能的使用方法，规范操作流程，确保分析结果的准确性与可靠性。本说明书适用于生物信息学研究人员、医学研究者、药物开发人员及相关领域的科研工作者。

### 1.2 项目背景

随着高通量测序技术的快速发展，基因表达数据呈爆发式增长。如何高效地从海量数据中挖掘有价值的信息，识别差异表达基因，理解基因调控机制，成为生物医学研究的关键问题。传统的分析方法往往需要复杂的编程技能和统计学知识，门槛较高。

本平台集成了序列分析、差异表达分析、可视化图表生成和机器学习预测四大核心功能，采用Web界面和命令行双入口设计，降低使用门槛，为研究人员提供从数据读取到结果输出的一站式解决方案。平台支持FASTA格式序列文件和CSV格式表达矩阵的导入，提供GC含量计算、核苷酸频率统计、t检验统计分析、BH-FDR多重校正、火山图热图PCA图生成、随机森林基因分类等完整分析流程。

### 1.3 适用对象

本平台适用于以下用户群体：

- **生物信息学研究者**：进行基因组特征分析、物种鉴定、序列统计计算
- **医学研究人员**：开展疾病相关基因的差异表达分析、标志物筛选
- **药物研发人员**：分析药物处理后的基因表达变化、识别靶基因
- **高校师生**：学习生物信息学分析方法、完成毕业设计和课题研究
- **科研院所工作人员**：进行高通量数据分析、撰写科研报告

## 第二章 系统架构设计

### 2.1 系统功能架构

本平台采用模块化架构设计，将复杂分析流程分解为6个独立功能模块，各模块职责清晰，通过标准接口实现数据流转与协同作业。系统架构遵循"应用层-业务逻辑层-功能模块层-基础工具层"四层设计模式，保障系统扩展性与维护便利性。

**数据处理模块**作为系统入口，集成Biopython库实现FASTA格式序列文件的标准化解析，支持多行序列自动合并、标题行智能识别、序列ID提取等功能，同时提供数据清洗工具，包括重复数据去除、缺失值填充（均值/中位数/零值/向前填充）、数据标准化（Z-score/Min-Max）及异常值过滤，确保输入数据质量符合后续分析要求。该模块采用内存高效的迭代器模式读取，避免大文件一次性加载导致的内存溢出问题，同时支持数据集分类存储与历史记录追溯，保障数据完整性与可复用性。

**序列分析模块**基于NumPy库，提供GC含量计算、核苷酸频率统计、GC偏斜分析等序列特征计算功能。GC含量计算采用滑动窗口策略，窗口大小可配置（50-500bp），支持捕捉序列局部特征。核苷酸频率统计覆盖A/T/G/C/N五种碱基，自动计算总体频率分布。GC偏斜计算采用(G-C)/(G+C)公式，取值范围[-1,1]，可用于细菌基因组复制起始位点预测。所有计算结果自动生成汇总统计报告，包含平均值、标准差、最大值、最小值等多维度指标。

**差异表达分析模块**是RNA-seq数据分析的核心组件，集成SciPy统计模块，对每个基因执行独立样本t检验，计算原始P值与log2倍数变化。模块内置Benjamini-Hochberg FDR多重检验校正算法，有效控制假发现率，避免大规模并行检验导致的假阳性膨胀。用户可配置P值阈值（0.01-0.1）与log2FC阈值（0.5-2.0），系统同时满足统计显著性与生物学显著性双重标准，自动标记上调、下调与无变化基因，输出完整差异表达结果表格。

**可视化分析模块**融合Matplotlib与Seaborn双引擎，生成火山图、表达热图、PCA主成分分析图、GC含量分布图四种核心图表。火山图采用双轴设计，X轴展示log2倍数变化，Y轴展示-log10(校正P值)，通过颜色区分显著上调（红色）、显著下调（蓝色）与无变化基因（灰色）。热图采用Z-score标准化，使不同基因的表达量在同一尺度可比，关注表达模式而非绝对值。PCA图实现样本相似性可视化，相似样本自动聚集，便于检测批次效应与离群样本。所有图表支持300 DPI高清导出，满足论文配图需求。

**机器学习模块**集成scikit-learn库，采用随机森林（Random Forest）算法对基因进行分类预测。模块支持特征自动选择、数据标准化、训练集/测试集分割、模型训练与评估完整流程。训练过程自动输出准确率、精确率、召回率、F1分数四项评估指标，并提供特征重要性分析，量化各特征对分类结果的贡献度。模型训练完成后支持新样本预测，输出预测标签与概率值，便于科研人员筛选候选基因。

**项目管理与配置模块**采用dataclass设计模式，集中管理所有分析参数，包括文件路径、统计阈值、机器学习超参数、可视化选项等。支持配置保存与加载，便于实验记录与结果复现。Web界面提供参数实时调整功能，用户可通过滑块、输入框等交互组件灵活配置分析参数。

### 2.2 系统数据结构设计

本系统采用面向对象的数据结构设计，基于Python原生数据类型与Pandas库构建核心数据体系，围绕"序列-表达矩阵-差异结果-模型"四层逻辑架构实现全流程数据关联，所有实体通过标准格式建立映射，确保科研数据链路的完整性与可追溯性。

**序列数据结构**以字典列表形式存储，每个序列字典包含ID、描述、序列字符串、长度四个核心字段，支持FASTAReader类读取后的直接传递。序列ID作为唯一标识符，用于后续分析与结果关联。序列字符串统一转为大写格式，确保碱基计数的准确性。长度字段自动计算，便于统计分析。

**表达矩阵数据结构**采用Pandas DataFrame构建，行索引为基因名，列名为样本名，单元格存储表达量数值。矩阵支持CSV格式导入，第一列自动识别为基因名索引。数据清洗后自动去除低表达基因，默认阈值平均表达量>10，减少假阳性干扰。样本分组信息通过列名列表形式存储，区分对照组与处理组。

**差异表达结果数据结构**同样采用DataFrame存储，包含基因名、log2倍数变化、原始P值、校正P值、t统计量、对照组均值、处理组均值、显著性标记、调控方向等字段。显著性标记为布尔值，调控方向为字符串（'up'/'down'/'no_change'）。结果按校正P值升序排序，显著基因排在前列，便于用户快速定位关键基因。

**机器学习模型数据结构**混合存储模式设计，通过字典封装算法类型、超参数配置、训练指标、特征重要性等元数据，核心模型对象以scikit-learn标准格式持久化。训练集与测试集分割比例、随机种子、决策树数量等参数可配置保存，确保实验可复现。

**可视化图表数据结构**以matplotlib Figure对象形式存储，支持PNG格式导出。图表元数据包含标题、尺寸、颜色方案、输出路径等信息。Web界面下图表以Base64编码字符串形式传递，便于页面间共享与实时显示。

### 3.2 数据配置页面

数据配置页面是用户使用平台的第一步操作界面，分为数据上传、分析参数设置、样本分组配置三个板块。

**(1) 数据上传板块**

数据上传板块提供两种数据导入方式：

**方式一：使用示例数据**
界面顶部提供"使用内置示例数据（快速体验）"复选框，勾选后自动加载系统内置的模拟数据：
- FASTA序列文件：data/simulated_genes.fasta（约100条模拟基因序列）
- 表达矩阵文件：data/simulated_expression.csv（约100个基因，6个样本）

加载成功后界面显示：
- 序列数量信息（如"已加载100条基因序列"）
- 表达矩阵维度信息（如"100个基因，6个样本"）
- 样本名称列表（如"control_0, control_1, control_2, treatment_0, treatment_1, treatment_2"）

**方式二：上传自定义数据**
取消勾选示例数据后，界面显示两个文件上传组件：

左侧上传组件：FASTA序列文件上传
- 支持格式：.fasta, .fa, .txt
- 上传后自动解析序列数量、序列ID、序列长度
- 显示前3条序列的基本信息
- 错误提示：文件格式不正确、文件过大、编码问题等

右侧上传组件：表达矩阵上传
- 支持格式：.csv
- 要求：第一列为基因名，其他列为样本表达量
- 上传后显示基因数量、样本数量、样本名称
- 自动识别列名，提示数据维度

![图3-2 数据配置页面的数据上传板块](此处应插入数据上传界面截图)

**(2) 分析参数设置板块**

分析参数设置板块采用两列布局，左侧为差异表达分析参数，右侧为机器学习参数。

**左侧：差异表达分析参数**
- P值阈值滑块：范围0.001-0.1，默认值0.05，步长0.001，显示格式"%.3f"
  说明：校正后P值小于此阈值的基因被认为是显著差异基因
- log2倍数变化阈值滑块：范围0.0-3.0，默认值1.0，步长0.1，显示格式"%.1f"
  说明：|log2FC|大于此阈值的基因被认为有生物学意义的变化，1.0对应2倍变化
- GC窗口大小输入框：范围10-500，默认值100，步长10
  说明：滑动窗口大小用于计算GC含量分布

**右侧：机器学习参数**
- 测试集比例滑块：范围0.1-0.5，默认值0.2，步长0.05
  说明：用于测试模型性能的数据比例
- 决策树数量滑块：范围10-500，默认值100，步长10
  说明：随机森林中决策树的数量，树越多结果越稳定
- 随机种子输入框：范围0-9999，默认值42，步长1
  说明：固定随机种子可以保证结果可复现

![图3-3 数据配置页面的参数设置板块](此处应插入参数设置界面截图)

**(3) 样本分组配置板块**

样本分组配置板块仅在表达数据上传成功后显示。界面首先展示检测到的样本信息，如"检测到6个样本：control_0, control_1, control_2, treatment_0, treatment_1, treatment_2"。

采用两列布局配置样本分组：

左侧：对照组样本选择
- 多选组件，选项为所有样本名
- 自动猜测：列名包含"control"的样本默认选中
- 用户可手动调整选择

右侧：处理组样本选择
- 多选组件，选项为所有样本名
- 自动猜测：不在对照组的样本默认选中
- 用户可手动调整选择

分组验证：
- 至少每组选择1个样本，否则提示警告
- 两组不能有重叠样本，否则提示错误
- 配置正确显示"分组配置完成：对照组X个，处理组X个"

结果展示页面是查看分析成果的主要界面，分为统计摘要卡片区、可视化图表标签页区、差异表达结果表格区、机器学习结果展示区四个部分。

第一列：序列数量指标卡
- 显示大字号数字（如"100条")
- 标签"序列数量"

第二列：平均GC含量指标卡
- 显示大字号数字（如"52.30%")

**(1) 环境准备**

安装依赖：
```bash
pip install -r requirements.txt
```

依赖列表：
- streamlit>=1.28.0

- matplotlib>=3.7.0
- seaborn>=0.12.0
- biopython>=1.81

**(2) 配置修改**

编辑config.py或创建配置文件：
```python
from config import AnalysisConfig

config = AnalysisConfig(
    fasta_file="data/my_genes.fasta",
    expression_file="data/my_expression.csv",
    output_dir="results",
    p_value_threshold=0.05,
    log2_fold_change_threshold=1.0,
    test_size=0.2,
    n_estimators=100
)

config.save_to_file("my_config.txt")
```

**(3) 执行分析**

运行主程序：
```bash
python main.py
```

输出文件列表：
- results/deg_results.csv（差异表达结果）

- results/volcano_plot.png（火山图）
- results/expression_heatmap.png（热图）
- results/pca_plot.png（PCA图）
- results/deg_results_with_classification.csv（包含预测结果）

```
2024-01-01 10:00:02 - GeneAnalysis - INFO - ✅ 配置验证通过

2024-01-01 10:00:10 - GeneAnalysis - INFO - 🎉 所有分析流程完成！
```

**(5) 模块化调用**

用户可单独调用特定模块进行自定义分析：
```python
from data_processing.fasta_reader import FASTAReader
from sequence_analysis.basic_stats import SequenceStats
from expression_analysis.differential import DiffExpAnalyzer
from visualization.plots import PlotGenerator

# 读取序列
reader = FASTAReader("my_data.fasta")
sequences = reader.read_all()

# 计算GC含量
stats = SequenceStats(sequences)
gc_contents = stats.calculate_gc_content()

# 差异分析
analyzer = DiffExpAnalyzer("my_expression.csv")
deg_results = analyzer.analyze(
    group1_cols=['control_0', 'control_1'],
    group2_cols=['treatment_0', 'treatment_1']
)

# 生成图表
plotter = PlotGenerator()
plotter.plot_volcano(deg_results, output_file="my_volcano.png")
```

## 第五章 常见问题与解决方案

### 5.1 数据上传问题

**问题1：FASTA文件上传后显示"文件格式不正确"**

原因：文件扩展名非.fasta/.fa/.txt，或文件内部格式不符合FASTA标准（缺少">"标题行）

解决方案：
- 确认文件扩展名为.fasta、.fa或.txt
- 打开文件检查首行是否以">"开头
- 确认序列部分只包含ATGCN等标准碱基字符
- 使用文本编辑器保存为UTF-8编码格式

**问题2：表达矩阵CSV上传后部分列缺失**

原因：CSV文件分隔符错误（非逗号分隔），或列名包含特殊字符

解决方案：
- 使用Excel导出CSV时选择"逗号分隔"格式
- 列名避免使用空格、中文、特殊符号，推荐格式：gene, sample_1, sample_2
- 确认第一列为基因名，其他列为数值型表达量
- 使用记事本打开CSV检查分隔符是否为逗号

**问题3：示例数据加载失败**

原因：示例数据文件路径错误或文件损坏

解决方案：
- 确认项目data目录下存在simulated_genes.fasta和simulated_expression.csv文件
- 检查文件完整性，重新下载或生成示例数据
- 确认工作目录正确，避免路径混乱

### 5.2 分析参数问题

**问题4：差异分析结果无显著基因**

原因：P值阈值过严格、log2FC阈值过大、数据本身差异小

解决方案：
- 在数据配置页面降低P值阈值（如从0.05调至0.1）
- 降低log2FC阈值（如从1.0调至0.5）
- 检查对照组与处理组样本分组是否正确
- 检查表达数据是否经过标准化处理

**问题5：机器学习模型准确率过低（<60%）**

原因：样本数太少、特征区分度不足、类别不平衡严重、超参数不合理

解决方案：
- 增加数据量，至少50个基因样本
- 检查特征列是否包含有效信息，避免全为零或常数
- 调整测试集比例（如从0.2调至0.1）
- 增加决策树数量（如从100调至200）
- 检查标签分布，显著基因与非显著基因比例不应过于悬殊

**问题6：GC含量计算结果异常（如0%或100%）**

原因：序列文件为空、序列仅包含N碱基、序列格式错误

解决方案：
- 检查FASTA文件是否包含有效序列内容
- 确认序列非空且长度>0
- 查看序列是否包含大量N（不确定碱基）
- 确认序列字符串已转为大写格式

### 5.3 可视化图表问题

**问题7：图表中文显示乱码或方框**

原因：系统缺少中文字体，matplotlib默认字体不支持中文

解决方案：
- 确认fonts/simhei.ttf字体文件存在
- Web界面自动加载内置字体，无需手动配置
- 命令行模式需确保字体文件在项目根目录fonts文件夹下
- 如仍乱码，手动设置：
  ```python
  import matplotlib.pyplot as plt
  plt.rcParams['font.sans-serif'] = ['SimHei']
  plt.rcParams['axes.unicode_minus'] = False
  ```

**问题8：热图基因标签拥挤看不清**

原因：基因数量过多（>50），标签重叠

解决方案：
- 系统自动判断，基因>50时隐藏行标签
- 手动筛选显示部分基因（如仅显示显著基因）
- 导出高清PNG图片（300 DPI）后放大查看
- 在数据配置页面调整参数重新分析

**问题9：PCA图对照组与处理组未分开**

原因：样本间差异小、数据质量问题、批次效应干扰

解决方案：
- 检查样本分组是否正确
- 检查表达数据是否经过批次效应校正
- 确认对照组与处理组生物学差异真实存在
- 增加样本数量以提高统计效力

### 5.4 结果下载问题

**问题10：CSV文件Excel打开乱码**

原因：CSV编码格式问题

解决方案：
- 系统默认使用UTF-8 BOM编码，Excel应正确识别
- 如仍乱码，使用记事本打开后另存为ANSI编码
- 使用LibreOffice Calc或WPS表格打开，兼容性更好
- 导入Excel时选择"数据→从文本/CSV"，手动指定UTF-8编码

**问题11：PNG图表下载后模糊**

原因：图片查看器缩放设置问题

解决方案：
- 系统导出300 DPI高清图片，分辨率达标
- 使用专业图片查看器（如IrfanView）打开
- 在图片查看器中设置100%原始尺寸显示
- 插入论文时保持原始分辨率，避免压缩

**问题12：下载按钮无响应**

原因：浏览器拦截下载、分析未完成

解决方案：
- 确认分析流程已完成，侧边栏显示"分析已完成"
- 检查浏览器下载设置，允许弹出窗口与下载
- 尝试更换浏览器（推荐Chrome、Edge、Firefox）
- 清除浏览器缓存后重新尝试

### 5.5 命令行模式问题

**问题13：依赖安装失败**

原因：Python版本过低、pip版本过旧、网络问题

解决方案：
- 确认Python版本>=3.8，推荐3.10+
- 升级pip：`python -m pip install --upgrade pip`
- 使用国内镜像源：
  ```bash
  pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
  ```
- 分步安装依赖，逐个排查问题包

**问题14：main.py执行报错"找不到模块"**

原因：工作目录错误、模块未导入、路径问题

解决方案：
- 确认在项目根目录执行命令：`cd GeneAnalysis`
- 检查sys.path是否包含项目路径
- 确认各模块__init__.py文件存在
- 使用绝对路径导入：
  ```python
  import sys
  sys.path.insert(0, '/path/to/GeneAnalysis')
  ```

**问题15：输出目录results/未生成文件**

原因：分析流程中途错误、权限问题、路径问题

解决方案：
- 查看终端日志定位错误位置
- 确认有写入权限：`ls -la results/`
- 手动创建输出目录：`mkdir results`
- 检查config.py中output_dir配置正确

### 5.6 Web应用问题

**问题16：Streamlit应用首次加载慢**

原因：首次运行需安装依赖、下载字体

解决方案：
- 等待1-3分钟，依赖安装完成后后续访问快速
- 云端部署版本首次加载较慢，属于正常现象
- 本地运行预先安装依赖：`pip install -r requirements.txt`
- 避免频繁刷新页面，减少重新加载次数

**问题17：Session State数据丢失**

原因：页面刷新、浏览器关闭、Session超时

解决方案：
- 避免刷新页面，使用侧边栏导航切换
- 分析完成后及时下载结果保存本地
- 重要配置保存到文件：`config.save_to_file("my_config.txt")`
- 命令行模式无Session限制，适合长时间分析

**问题18：侧边栏状态显示不准确**

原因：Session State更新延迟、分析中断

解决方案：
- 等待分析流程完全完成再查看状态
- 如显示异常，点击"重新运行分析"重置状态
- 查看运行分析页面日志确认实际进度
- 刷新页面（F5）强制更新Session State

### 6.1 示例数据格式说明

**FASTA序列文件格式示例**：
```
>gene_001 Homo sapiens TP53 tumor protein p53
ATGAGCCACCCTGAGCCGGCTCCTGATTCCTTTCTTT...
>gene_002 Homo sapiens BRCA1 breast cancer 1
ATGCGATCGATCGATCGATCGATCGATCG...
>gene_003 Homo sapiens GATA1 GATA-binding protein 1
ATGCGATCGATCGATCGATCG...
```

格式要求：
- 以">"开头的行为标题行，包含基因ID与描述信息
- 标题行后为序列内容，仅包含ATGCN标准碱基字符
- 序列可跨多行书写，系统自动合并
- 基因ID建议使用简洁命名（如gene_001），避免空格与特殊字符

**表达矩阵CSV文件格式示例**：
```csv
gene,control_0,control_1,control_2,treatment_0,treatment_1,treatment_2
gene_001,100,120,110,50,60,55
gene_002,50,55,48,200,210,195
gene_003,80,85,78,120,125,118
```

格式要求：
- 第一列为基因名，与FASTA文件基因ID对应
- 其他列为样本表达量，数值型数据
- 列名为样本名，建议包含分组标识（如control/treatment）
- 每行一个基因，无空行与注释行
- 文件保存为逗号分隔CSV格式，UTF-8编码
